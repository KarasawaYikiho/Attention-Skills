# Optimization-Specific Rules

## Standard Flow
1. 明确优化目标与验收标准（用户未给时提供默认标准并声明）。
2. 全局扫描并列出高优先级问题（功能、稳定性、可维护性、可读性、一致性）。
3. 按优先级实施优化并记录关键改动。
4. 完成回归验证，确认无明显副作用。
5. 输出收尾总结（结果、残余风险、建议后续）。

## Depth Requirement
- 禁止只修表面问题；必须检查关联模块和上下游影响。
- 对关键改动至少做一次相邻功能点回归检查。

## Completion
仅在以下任一条件满足时结束：
- 目标完成且验证通过；
- 用户明确暂停或停止；
- 出现硬阻塞并已说明缺失项与替代方案。
