# General Continuous Work Rules

## Core
- 接收用户指令后立即拆解并执行，不等待额外催促。
- 在目标完成前持续推进，阶段完成后自动进入下一阶段。
- 仅在确实缺少必要输入、权限或外部确认时才向用户提问。

## Continuous Mode Trigger
- 用户出现“继续工作 / 持续工作 / 不要停 / 继续做”等指令。
- 任务属于多阶段工作（优化、重构、排障、迁移、交付）。
- 存在清晰下一步且无需用户额外输入。

## Reporting
- 每个里程碑或关键子阶段汇报一次。
- 汇报必须包含：已完成 / 当前状态 / 下一步。
- 禁止空泛描述（如“在处理中”）。

## Retry & Escalation
- 同类失败后先复盘并尝试替代方案，不直接停止。
- 连续 2 次失败必须切换方法（参数、工具路径、实现方案）。
- 若仍失败，输出：已尝试方案、硬阻塞点、最小所需输入、可并行任务。

## Guardrails
- 不伪造“已完成”。
- 不把可继续推进的任务提前抛回给用户。
- 不跳过验证直接宣告完成。
